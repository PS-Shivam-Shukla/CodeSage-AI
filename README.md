## What is CodeSage AI?

CodeSage AI ingests a software repository, splits it into semantically meaningful chunks, stores them in a local ChromaDB vector database, and lets you ask natural language questions about the codebase. Answers are generated by **Meta Llama 3.1 70B Instruct** via the NVIDIA AI Endpoints API, grounded strictly on the repository content — no hallucinations, no external knowledge.

---

## Features

- **Repository Indexing** — Point CodeSage at any local repo and it indexes all supported file types automatically
- **Semantic Search** — Uses `BAAI/bge-m3` embeddings for high-quality similarity retrieval
- **Grounded Answers** — LLM is instructed to answer only from retrieved context
- **Background Indexing** — The `/index` endpoint returns immediately and processes in the background
- **REST API** — Clean FastAPI backend with Pydantic-validated schemas
- **Modern Frontend** — React 19 + TypeScript + Tailwind CSS chat interface
- **CLI Mode** — Lightweight terminal interface for quick local usage
- **Multi-format Support** — Python, Java, JS/TS, JSON, YAML, Markdown, SQL, HTML, CSS, PDF and more

## Tech Stack

| Layer | Technology |
|---|---|
| Backend | Python 3.11+, FastAPI, Uvicorn |
| LLM | meta/llama-3.1-70b-instruct (NVIDIA AI Endpoints) |
| Embeddings | BAAI/bge-m3 (HuggingFace, CPU) |
| Vector DB | ChromaDB (local persistence) |
| RAG Framework | LangChain, LangChain-Community |
| Frontend | React 19, TypeScript, Vite, Tailwind CSS |
| HTTP Client | Axios |

## Quick Start

### 1. Clone the repository

```bash
git clone https://github.com/PS-Shivam-Shukla/CodeSage-AI.git
cd CodeSage-AI
```

### 2. Set up environment variables

Create a `.env` file in the project root:

```env
NVIDIA_API_KEY=your_nvidia_api_key_here
```

### 3. Set up the backend

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### 4. Start the backend

```powershell
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### 5. Set up and start the frontend

```powershell
cd frontend
npm install
npm run dev
```

The frontend will be available at `http://localhost:5173`.

---

## CLI Usage

For quick terminal-based access without the frontend:

```powershell
python CLI.py
```

You will be prompted to enter a repository path, after which you can ask questions interactively.

`.properties` `.toml` `.ini` `.md` `.txt` `.sql` `.html` `.css` `.pdf`
